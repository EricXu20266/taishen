## 一、缘起

2026年五一节，开发者还在短视频平台教人怎么使用Claude Cowork接入deepseek。

一周之后，他在想为啥不自己做一个。

但这个想法让他犹豫。市面上 Agent 已经够多了，AI 更新速度太快，一个个人开发者能做出什么？

万事不决，交给玄学！

2026年5月8日，他打开了 DeepSeek 的对话窗口。输入了三个数字：**414**。

deepseek赛博算命了下。

按照梅花易数，4 为震（雷），1 为乾（天），动爻第四位。得本卦**雷天大壮**、互卦**泽天夬**、变卦**地天泰**。

大壮卦第四爻辞：「贞吉，悔亡。藩决不羸，尚往也。」——守正得吉，冲破藩篱，可以前行。

变卦地天泰：「天地交泰，万物通达。小往大来，吉亨。」

泰卦，天地定位之后的第一卦，排在第十一卦，却是所有卦里最通达、最和谐的一卦。

这个字既是卦象的终点，也恰好暗合他想做的事：让 DeepSeek 的能力（深）与普通人的需求（通达）交汇。他给项目取名**泰深**，英文名 **AllinDeepseek**——孤注一掷的决绝，配上地天泰的从容。

这份白皮书所依托的每一行代码，都始于那三个数字和一句爻辞。泰深不是又一个"DeepSeek 客户端"。它是一个人用卦象确认方向后，从零开始打造的、为普通人而生的 AI 工作台。

## 二、现在：AI 交付平台

泰深当前版本 v1.3.6，是一款完整的桌面应用（Windows / macOS），深度适配 DeepSeek V4 系列模型。

在 awesome-deepseek-agent 官方收录的 23 个工具里，15 个是终端命令行，3 个是 VS Code 插件。绝大部分面向开发者——敲命令、写代码、调 API。泰深是其中仅有的两个 GUI 桌面应用之一，但它和另一个 GUI（Cherry Studio）的定位完全不同：Cherry Studio 是「多模型聊天客户端」，泰深是「Agent 工作台」。聊天是手段，交付是目的。

横向拉一遍列表，泰深有四个能力是独有的：**主动弹窗引导**（其他工具等你写好 Prompt）、**手机远程控制桌面 Agent**（微信/飞书/QQ 接入，出门时手机发需求，桌面端持续产出）、**一键生成完整文档**（Word/PPT/PDF/Excel，而非仅输出文本）、**DeepSeek 前缀缓存深度优化**（99% 命中率，长会话成本几乎不增长）。

它不追求「无所不包」，而是聚焦一个核心命题：**从信息到成果**。

### 2.1 我会主动问你，不用你学会问我

大多 AI 工具假设你会写 Prompt——问题越精确，回答越靠谱。泰深不这么假设。

我是桌面里的 GUI 智能体，面向的是普通用户。你不会写 Prompt？说一句「帮我分析下这几份竞品报告」就行。需求模糊？我会弹窗问你：「对比维度你更看重功能还是定价？」「输出 Word 还是 PPT？」——直到确认你的真实意图。

`ask_user_question` 弹窗是我的核心交互方式，不是辅助功能。一次弹窗确认，省去三轮回滚。用户点一下按钮，比猜错方向重做快一百倍。

确定了目标之后，剩下的我来。底层引擎是**规划 → 拆解 → 执行 → 验证 → 交付**五个阶段，一条线走到底。右侧面板实时展示每一步进展——你知道我在干什么，但不需要告诉我怎么干。关键决策点继续弹窗确认，安全边界内我自己做主。

用户的角色不是 Prompt 工程师，不是操作员，是**委托人**。

### 2.2 普通人能用的 DeepSeek

安装 → 填入 API Key → 拖入文件 → 开始干活。用户不需要知道 token、temperature、思考强度。需要做竞品分析？拖入几份报告，一句话描述需求。需要写论文？拖入参考文献，告诉 AI 你的研究方向。

泰深把 DeepSeek 的能力封装成了直觉化的产品体验——这是目前 awesome-deepseek-agent 列表中其他 CLI 工具做不到的事。

### 2.3 DeepSeek 原生深度整合

泰深不是通过 OpenAI 兼容层转译 DeepSeek。它使用 DeepSeek 原生的 `deepseek` 思考格式，直接对接推理 API，完整释放 V4 Pro 的深度思考能力。中间没有损耗层。

同时，泰深并非「绑定」DeepSeek。它内置了**多模型路由**能力——DeepSeek 是主引擎，但特定子任务（如视觉识别、轻量代码修补）可以动态分配到其他模型。用户不需要了解哪个模型擅长什么——Agent 根据任务类型自动匹配最优模型。更进一步，FlexDog 动态模型引擎让用户在对话中随时切换模型，不需要重启会话：想用 V4-Pro 深度思考一个问题，下一句话切到 V4-Flash 快速响应，即时生效。这种架构在保持 DeepSeek 核心地位的同时，为未来模型生态的演进留出了空间。

更深层的整合体现在消息流的设计上。基于 DeepSeek 的前缀缓存机制，泰深自主设计了 System Prompt 的消息流组装策略：将高频不变的系统指令前置，低频变化的用户输入和工具结果后置。这不是配置参数的调整，而是对整个 System Prompt 结构的重新编排。

效果是什么？在千万 token 级别的编码会话中，**缓存命中率稳定在 99%**。就在此刻，你正在阅读的这份白皮书所依托的泰深会话，**缓存命中率是 98.7%**。这意味着每轮对话的 API 成本几乎不随会话长度增长——你可以连续工作数小时、处理数十万字的资料，而费用保持不变。在大量开源复刻项目中，这样的命中率并不多见。

### 2.4 跨场景覆盖

**桌面端**：完整的 GUI 工作台，多会话并行，文档拖入解析，一键生成 Word/PPT/PDF/Excel。

**手机端**：通过飞书、微信、QQ 任一 IM 接入。流式回复、文件传输、审批弹窗全部打通。出门时用手机发一条需求，桌面上的泰深持续为你产出。

这是 awesome-deepseek-agent 列表中唯一支持 IM 远程接入的桌面应用。它背后的意义是：AI 不应该只活在终端里，它应该在你工作流的每一个触点。

### 2.5 上下文与记忆

任何用过 AI 的人都知道一个痛点：聊着聊着 AI 就忘了前面说过什么。

泰深在上层，通过持续跟踪会话中的关键决策、用户偏好和项目上下文，确保 AI 在多轮对话中保持连贯。它不是简单的「记住对话」，而是理解什么是重要的、什么该遗忘。

在底层，跨会话的记忆系统让 AI 记住你的身份、工作风格和常用工具。30 天半衰期的自适应衰减机制，确保记忆不会无限膨胀。你的写作偏好、代码风格、甚至对某些工具的倾向——AI 越用越懂你。

在上下文层面，泰深内置了**智能压缩引擎**。长会话中，当对话历史接近模型窗口上限时，系统自动压缩历史消息——保留关键决策和重要信息，精简冗余细节。冷启动重入会话时，系统注入最大 150K token 的历史摘要，让你立刻衔接上次进度。这意味着你可以连续工作数小时，AI 不会「失忆」，也不会因为上下文爆满而无法继续。

除了记忆，泰深还有一套**三层宪法系统**，让用户以自然语言定义 AI 的行为准则。L1 全局宪法——对所有项目和会话生效，比如「回答用中文」「不主动生成 docx」。L2 项目说明——限定在当前项目内，比如「这是 DeepSeek 社区 PR 项目，风格正式」。L3 会话覆盖——临时调整，会话关闭即失效。三层规则按优先级合并注入到每轮对话的 System Prompt 中，用户不需要写 prompt 模板——用自然语言说一次，泰深就记住。宪法系统和记忆系统的区别在于：记忆是 AI 观察到的关于用户的被动事实，宪法是用户主动下达的主动规则。两者互补，构成了 AI 行为可控性的「软硬两手」。

除了这些核心系统，泰深还内置了一套实用基础设施：**定时任务调度器**让 AI 在指定时间自动执行任务——比如每天早晨汇总行业新闻；**回收站系统**让误删的会话和 Skill 随时捞出，永不过期；**中英文双语界面**一键切换，覆盖全部 UI 和系统提示词。这些不是噱头，而是完整工作流体验的必备拼图。

### 2.6 四层安全防线

能力越大，边界越重要。泰深的安全体系不是简单的「限制 AI」，而是一套让用户可以放心把更多权限交给 AI 的**信任基础设施**。四层防线，纵深防御：

**第一层：路径访问控制（PathGuard + PathResolver）**。AI 只能访问用户明确挂载的工作区目录。系统目录（C:\Windows、/etc）、磁盘根目录、用户主目录均在拦截列表中。即使 AI 尝试用短文件名（PROGRA~1）或符号链接绕过，PathGuard 也能识别并阻止。PathResolver 将 AI 使用的虚拟路径映射到物理路径，双重验证，防止路径逃逸。

**第二层：命令执行控制（Sandbox）**。三层沙箱模型——L1 严格审批（每步弹窗确认）、L2 自主审查（AI 自行判断安全性）、L3 完全信任。致命命令直接拦截（`rm -rf /`、`shutdown`、`chmod 777` 等），不进入审批流程。命令替换注入（`$()`、反引号）和脚本引擎内联执行（`node -e`）也在沙箱层被阻断。

**第三层：网络边界控制（NetworkGuard）**。默认开放 localhost、DeepSeek API、GitHub 等常用域名。访问其他域名需向用户申请白名单。内网地址（127.x、192.168.x、10.x）自动拦截，防止 SSRF 攻击。

**第四层：门控防重试（GateRejectionTracker）**。同一操作被用户拒绝 2 次后，系统自动阻止，AI 不再弹窗打扰。这不是限制用户的选择权，而是防止 AI 在用户明确表达拒绝意向后反复纠缠。

每次文件修改自动本地备份，随时恢复。安全边界不是为了束缚 AI——恰恰相反，**正是因为有了这四层防线，用户才敢把更多权限交给 AI**。

### 2.7 AI 自诊断：不出问题不需要人

所有 AI 工具都会出错——关键是出问题之后谁来查、怎么查。传统模式是用户翻 Console、截日志、发给开发者。泰深的思路不同：**AI 应该能自己给自己「看病」。**

泰深内置了 6 级（trace/debug/info/warn/error/fatal）× 9 分类（agent/tool/session/security/memory/system/network/ui/remote）的结构化日志系统。日志独立存储在 `taishen-logs.db`，操作日志保留 3 天，执行追踪保留 7 天。更重要的是——AI 可以直接调用 `log_query` 工具查询自己的日志：

- 工具调用失败了？AI 查 error 日志定位根因，几秒内给出诊断
- 会话保存异常？AI 按 sessionId 过滤日志，追踪完整调用链
- 性能下降？AI 查 trace 记录，找到耗时瓶颈

这不是事后调试工具，而是让 AI 具备**运行时自我感知**的基础设施。对用户来说，出问题时不需要懂技术、不需要翻日志——AI 自己诊断、自己修复、自己汇报。从「用户帮 AI 排错」到「AI 自己排错」，这是一个质的跨越。

### 2.8 泰案画布：AI 的流式工作空间

前面讲的是 AI 怎么交互、怎么安全、怎么自愈。但还有一个根本问题没有解决：**AI 工作时你看不到过程。**

传统的 AI 对话是一个黑箱——你发需求，AI 回结果。中间发生了什么，你不知道。泰案（Tai An）改变了这一点。它是泰深内置的流式内容工作空间，一个 AI 与你共享的画布。AI 在上面边写边呈现，你实时看到内容成型——不是等着拿最终结果，而是看着它被打造出来。

六种画布覆盖全场景：

**写作画布**：AI 逐字逐句地写，你逐字逐句地看。适合长文、报告、文案——任何需要边看边调整的创作场景。不是「AI 写完你再改」，是「AI 写的时候你就可以介入」。

**代码画布**：语法高亮 + 内嵌终端。AI 写完代码，在同一窗口内跑编译、跑测试。你看着它写代码、发现报错、自己修、再跑——全程不需要切窗口。

**HTML 画布**：继承内置预览器的所见即所得能力。点击框选页面元素即可标注，设计评审不再靠截图圈圈画画。

**终端画布**：AI 的命令执行全程可见。你知道它在做什么，也可以随时叫停。

**数据分析画布**：表格 + 四种图表。拖入 CSV 直接出图，AI 帮你清洗数据、选择可视化方式。

**图片画布**：展示 AI 生成的图片、截图，支持标注和对比。

关键设计：所有画布都是**双向编辑**——AI 可以写，你也可以直接在上面改。历史版本追踪精确记录每一次变更，随时回退到任意版本。你不会因为「AI 改了一版我不喜欢」而丢失之前的内容。

画布不是又一个编辑器。它解决的是一个本质问题：**信任。** 当 AI 的工作过程从黑箱变成可见，你不需要「相信 AI 会做对」——你看着它做。

独立于画布系统之外，v1.3.6 的内置浏览器也已升级为**独立窗口**——从主会话剥离，拥有完整的工具栏（代理支持、多标签管理、收藏夹、下载防呆），不再模态冻结主窗口。一键「Copy 设计」把网页存到工作区，所见即所得。

### 2.9 MCP 开放生态：能力随生态生长

泰深的能力不是封闭的。它原生支持 MCP（Model Context Protocol）开放协议——一套让 AI 接入外部工具服务的标准接口。MCP 就像 AI 世界的「USB-C 集线器」：任何遵循 MCP 标准的第三方服务，都可以插入泰深，成为 AI 可调用的新工具。

目前已接入的 MCP 服务包括：代码图谱（符号级代码索引，精准定位函数和调用链，让 AI 在大型项目中不迷路）、浏览器深度控制（Chrome DevTools 协议，不止于截图——性能分析、内存快照、网络请求全链路追踪）、深度搜索与内容提取（跨 22 个垂直领域的结构化搜索，一天 1000 次免费调用）。这些不是泰深自研的功能，而是通过 MCP 生态「长出来」的能力。

对用户来说，这意味着泰深的能力边界不取决于开发团队——取决于整个 MCP 生态的演进。新的工具、新的数据源、新的垂直领域，只要生态在生长，泰深的能力就在生长。

### 2.10 分身并行：一个 AI，多条战线

传统 AI 助手一次只能做一件事。你说「帮我查 A、B、C 三家公司的财报，再对比分析」——它得一个一个查，串行等待。

泰深用空间换时间。当你给它多个独立任务时，它可以同时分身——让多个工作线程并行开工，A 查财报的同时 B 读行业报告、C 跑数据对比，各自独立完成后汇总到主控台。用户感知到的效果是：原本需要五分钟的串行任务，现在一分钟就能拿到全部结果。

更关键的是，每个分身可以调用不同的模型。泰深原生支持多 Provider——DeepSeek 是主引擎，但当你需要识别一张图片而 DeepSeek 当前不支持视觉时，一个分身可以自动走 GPT-4o 看图，另一个分身继续用 DeepSeek 深度分析文字。不同模型各展所长，在同一个任务中无缝协作。对用户来说，你只需要说「帮我分析这几张截图」，不需要知道哪个模型能看图、哪个模型擅长推理——泰深自动把任务拆给最合适的分身，各自用最擅长的模型完成。

这不是噱头。需要同时在三个平台上调研同一家公司、同时分析五份竞品报告、或者一边写代码一边查文档一边跑测试——分身并行把 AI 从「单线程」升级为「多线程」。用户不需要关心底层怎么调度，只需要知道：**给了多个任务，它们会同时推进。**

## 三、已实现：从工具使用者到经验封装者

泰深的交付平台底盘已经稳固，v1.3.6 画布系统上线后进一步补齐了「可见性」这一环。经验封装系统在 v1.3.5 中已正式上线。

让我们退一步思考。当前所有 AI 工具——包括泰深——都遵循同一个模式：人类给 AI 配备了一组预设工具（文件读写、网页搜索、Shell 执行），AI 在这组工具内完成用户的任务。AI 是一个「工具使用者」。

但泰深已经在这条边界上跨出了决定性的一步。v1.3.5 版本中，经验封装系统从设计走向落地，构成一个从被动使用到主动创造的完整光谱：

**skillCreator** — AI 可以将一套成功的操作流程封装为 Skill。Skill 是 Markdown 工作流模板，遵循 agentskills.io 开放标准，创建后即时生效。它轻到几乎零成本——AI 注意到「查论文 → 提取摘要 → 生成文献综述」这个模式反复出现，几分钟就能封装成一个 Skill，下次用户一句话触发。

**skillPatcher** — Skill 不是一次性的。AI 在使用过程中发现某个步骤可以优化、或者需要覆盖新的边缘情况，直接调用 skillPatcher 追加或替换 Skill 内容。这是一条自我改进回路——Skill 越用越精准。

**tool_creator** — 当 Skill 不够用时，AI 可以编写真正的 TypeScript 工具。编译、注册、下次会话生效。适合需要调用外部 API、引入 npm 包、或执行特定计算逻辑的场景。

这三个工具在泰深发布最初版本就已经就位。但它们有一个共同的前提：**AI 在等待用户的指令。** 用户得先意识到「我需要封装这个」，AI 才会动手。

真正的质变在下一步：**触发权从用户转移到 AI。** 经验封装系统的后台引擎已在 1.3.5 中打下基础——行为指纹提取、模式检测、决策引擎三通道并行，为 AI 自主察觉摩擦做好了技术储备。

### 3.1 不是等待指令，是察觉摩擦

用户不会说「请帮我创建一个工具」。用户只会觉得某件事做起来很麻烦——每次处理同一个 API 的返回数据都要手动写 awk，每次发布短视频都要把 Markdown 逐平台改格式，每次查论文都要重复同一套检索逻辑。

用户感受到的是摩擦。摩擦不会自己报出名字。

泰深的下一步，是让 Agent 具备**操作指纹识别**能力。每次会话结束后，系统提取一组轻量指纹：用了哪些工具、按什么顺序、处理了什么格式的数据、用户说了什么触发词。当同一指纹在多次会话中重复出现，系统内部响起一个信号：**这个模式熟了，该封装了。**

### 3.2 封装有轻重：Skill 优先

不是所有重复模式都需要 TypeScript 代码。大多数只是固定的工作流步骤。Skill 是首选封装形态——轻量、即时生效、用户可以一眼看懂它在做什么。TypeScript 工具是兜底方案——仅在 Skill 无法覆盖（需要外部 API、特定计算逻辑、npm 依赖）时才走 tool_creator 路径。

对用户来说，不管是 Skill 还是工具，界面上的感知是统一的：**「我的泰深多了一个能力」**。

### 3.3 分层自治：什么自己做，什么必须问

AI 自主封装，避不开安全边界。一刀切的「每次审批」会扼杀体验，一刀切的「完全自主」会引入风险。答案是按风险分层：

- **纯数据变换**（格式转换、文本处理、字段映射）→ 静默创建，通知栏提醒。不接触外部世界，风险为零。
- **网络/文件操作**（调用外部 API、读写特定格式）→ 弹窗预览，一键确认。让用户知道多了什么，但不卡流程。
- **系统级操作**（执行命令、修改配置）→ 完整审批，展示源码。触碰安全边界的事，必须用户看过点头。

层级的划分不由 AI 自己判断——由封装内容的输入输出参数自动归类：有网络调用 → 第二层，有 shell 调用 → 第三层，纯数据变换 → 第一层。

### 3.4 能力会老，也会淘汰

如果 AI 一直在封装经验，三个月后用户会有几十个 Skill 和工具，其中一半只用了两次。

需要一套**生命周期管理**：每次调用记录时间戳和频次；30 天无人使用自动禁用（不删除，防止误杀）；每季度弹一张卡片提醒清理；AI 在封装前先检查——已有能力能不能覆盖这个模式？能不能给已有 Skill 加个参数替代新建？

### 3.5 这意味着什么

每个用户的泰深会逐渐长出不同的能力形态。不是靠 AI 偷偷造工具堆出来的——而是靠**察觉摩擦 → 封装经验 → 沉淀能力 → 持续改进**这个循环自然生长出来的。

做学术研究的人，Agent 会先封装一套文献检索 Skill，如果还不够用（需要调用特定数据库 SDK），再创建 TypeScript 工具。做内容创作的人，Agent 会把「Markdown → 各平台格式」封装成 Skill，发现 B 站需要特定 API 上传时，再补一个工具。

更重要的是，skillPatcher 让这些封装不是死的。下个月用户换了投稿平台，AI 能自己把 Skill 里的平台规则更新掉。工具也跟着用户的成长而成长。

AI 不再只是「回答你的问题」，而是主动察觉你的工作流中哪里有阻力，然后用最轻的方式把阻力消掉。

从「交付成果」到「封装经验」——这是泰深正在发生的进化。v1.3.5 经验封装系统已上线，下一步是让 AI 从「等待指令」走向「主动察觉」。工具不是目的，**经验的沉淀和复用才是。**

## 四、邀约

泰深选择 DeepSeek 作为唯一的完整适配供应商，不是权宜之计。是因为 DeepSeek V4 的推理深度、1M 上下文窗口、前缀缓存机制，恰好构成了这个 Agent 形态的底层支撑。

我们邀请 DeepSeek 团队：

### 4.1 评估泰深作为桌面端标杆

目前 awesome-deepseek-agent 列表以终端 CLI 和编辑器插件为主。一款面向普通知识工作者的 GUI 桌面工作台——能交付、能跨场景、为 DeepSeek 深度调优——是这个列表尚未覆盖的品类。

### 4.2 对 DeepSeek 生态的价值

- **降低使用门槛**。泰深让不需要敲命令行的用户也能用上 DeepSeek。
- **创造高留存场景**。学术研究、内容创作、软件开发——不是「试试看」的一次性体验，是形成工作流依赖的日活场景。
- **API 优化反馈**。泰深对 thinking format、缓存行为、峰谷定价的深度使用，为 API 团队提供真实生产环境的反馈数据。

### 4.3 关于闭源

泰深的核心组件未开源。我们尊重开源社区（Skill 系统兼容 agentskills.io 开放标准，MCP 协议支持全生态），但 Commander 引擎、消息流编排策略、进化算法等核心资产，是泰深区别于通用聊天客户端的关键壁垒。

这是商业软件的合理选择——就好比 Photoshop 不开源，但它对图像处理标准的推动仍然巨大。
