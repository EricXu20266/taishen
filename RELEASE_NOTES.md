## 🚀 泰深 v1.2.9 正式发布

经济适配模式 A 测上线，Fork 分身智能省钱。浮动峰谷时钟、分组持久化、PathGuard 加固等体验全面升级。

### 核心功能
- DeepSeek V4 全模型支持（V4-Pro / V4-Flash）
- 多供应商支持（DeepSeek / OpenAI / Mimo 等自由切换）
- 视觉代理 Myeyes — 模型不支持多模态也能"看图"，IM 渠道全面打通
- FlexDog 动态模型引擎 — 对话中随时切换模型
- Commander 形态 — AI 自主规划、调度、验证一条龙
- HTML 高级预览器 + 元素批注系统 — 所见即所得，选中即标注
- 回收站系统 — 误删会话、Skill、子代理随时捞出，永不过期
- Skill 自定义安装与扩展（兼容 agentskills.io 开放标准，支持 subgroup 子分组）
- MCP 服务接入（多源连接，支持 Streamable HTTP / SSE / stdio）
- IM 远程接入（飞书 / QQ / 微信），全面支持图片发送
- SubAgent 子代理并行调度（v3.1 架构，独立 Skill/Memory/MCP）
- 定时任务调度器 + 全局会话搜索
- macOS 双架构正式支持（x64 + arm64）

###v1.2.9

- A测更新，经济适配模式上线。一线操作员 Fork 分身自动注入经济模型，在高频任务中智能选择更省钱的模型配置，token 开销精细化管理。
  - 首次使用需要在设置-大模型配置，Provider具体的配置页里打开下配置，在选择好对应的子代理模型。
  - 开启后前台会话页的选择模型会变色，作为提醒。
  - 本次A测功能，没有使用强制手段，暂时靠AI自觉选择。
  - 你需要注意的是，Fork 分身不是无脑使用的，以下占优分析：
	  
	**Fork 分身 vs 主身直接执行 — 占优边界：**

	| 场景 | 主身 | Fork 分身 | 结论 |
	|------|------|----------|------|
	| 单工具调用 | ~3K tokens（缓存命中增量） | ~20K tokens（启动税） | 主身完胜 |
	| 3+ 工具调用 | ~15K tokens（3 轮增量） | ~25K × 1/3 Flash 单价 | 费用 Fork 胜 |
	| 并行独立任务 | 串行 6 轮 | 并行 max(3 轮) | 时间 Fork 胜 |
	| 上下文近上限 | tool_result 挤占主上下文 | 侧链隔离，不污染 | 保护预算 Fork 胜 |

	> Fork 只继承最后一条 assistant 消息，不含完整会话历史 → 启动税 ~15K-20K 恒定，不随会话增长。

- 会话列表分组持久化，同时在会话列表的文件夹组后点击"+"号打开对应的新会话面板。  
- 添加了一个浮动时钟组件，全局拖拽的峰谷定价时钟，玻璃面板 + 主题自适应配色，随时一眼看到当前时段处于 DeepSeek 的哪个定价区间——高峰还是低谷，心中有数。
- 子代理缓存命中率现在可以在会话预算面板直接看到，一线操作员 token 利用率一目了然。
- 项目上下文自动持久化重构，现在泰深再次进入项目文件夹时自动加载项目说明，无需重复探索项目结构。
- 调整了项目配置页窗口，编辑自动保存，关闭增加防误触确认。
- Skill 管理支持分组和子分组重命名，点一下铅笔图标直接改。
- 子 Agent 会话页新增状态指示器，消息流卡片实时同步。
- PathGuard 现在会正确拦截 git checkout 和 git restore 对工作区的破坏性操作，避免手滑丢代码。
  - 每当出现这种奇奇怪怪的更新条目时，就说明开发者中招了，痛苦！
- 停止按钮不再把最后一条消息恢复到输入框，思考时间格式统一为 H:MM:SS。
- 修复了一些小毛病。
